# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Vietnamese-language pipeline that (1) surveys YouTube popularity of keywords about *Quan Vũ / Quan Vân Trường / Quan Công / Quan Thánh*, and (2) runs sentiment analysis on comments of a fixed playlist (`PLlvlc45o3QQcwtas1taX7VlLyZ6Advf_3`). All user-facing text (logs, CSV docs, reports) is in Vietnamese — preserve this when editing.

## Environment & commands

Requires Python 3.10+ và (tuỳ chọn) `PLAYLIST_ID` trong `.env`. **Không cần** YouTube Data API key — pipeline dùng `yt-dlp` + `youtube-comment-downloader`.

```bash
python -m venv .venv
source .venv/Scripts/activate    # Git Bash on Windows
pip install -r requirements.txt

# Run full pipeline (step1 → step2 → step3 → visualize)
python src/run_all.py

# Or run a single stage (each script is standalone, no CLI args)
python src/step1_keyword_stats.py
python src/step2_crawl_comments.py
python src/step3_sentiment.py
python src/visualize.py
```

There is no test suite, linter config, or build step. `data/` and `report/` are generated by the pipeline.

## Architecture

Four sequential stages passing state through CSV files in `data/`:

```
step1_keyword_stats → data/keyword_*.csv, keyword_summary.csv
step2_crawl_comments → data/playlist_videos.csv, comments_raw.csv
step3_sentiment      → data/comments_labeled.csv, sentiment_report.csv
visualize            → report/*.html, *.png
```

Stages only communicate via the files above — `run_all.py` just `subprocess.call`s each script in order and aborts on non-zero return. Adding a stage means adding a script + an entry in `STEPS`, not wiring imports.

`src/youtube_client.py` centralizes data access without API key: wraps `yt-dlp` (`search_videos`, `get_video_stats`, `list_playlist_videos`) và `youtube-comment-downloader` (`iter_comments`). Mọi truy cập YouTube nên đi qua module này để giữ retry/backoff đồng nhất (`safe_call`).

`src/step3_sentiment.py` loads PhoBERT (`wonrax/phobert-base-vietnamese-sentiment`) via `transformers`. First run downloads ~500 MB; auto-detects GPU. Text goes through `clean_text` (strip emoji/URL/HTML) then `underthesea.word_tokenize` before classification — the tokenizer-expected input is the space-joined underthesea output (e.g. `"tôi yêu_nước"`), not raw text.

## Key constraints when changing code

- **Không có quota cứng** (đã bỏ YouTube Data API). Tuy nhiên `yt-dlp` gọi từng video, nên tránh tăng số keyword/playlist lớn quá mức để không bị bot-detection. Xem `docs/QUOTA.md`.
- **Comment crawl** trong `step2_crawl_comments.py` dùng `youtube-comment-downloader`, mặc định lấy tối đa `MAX_COMMENTS_PER_VIDEO=2000` top-level mỗi video (bỏ replies). Lỗi comments tắt được bắt ở `iter_comments` — đừng convert thành hard failure.
- **Configuration hotspots**: keyword list → `KEYWORDS` in `step1_keyword_stats.py`; playlist → `PLAYLIST_ID` env var / fallback in `step2_crawl_comments.py`; sentiment model → `MODEL_NAME` in `step3_sentiment.py`.
- **CSV schemas** are consumed by downstream stages and documented in `docs/ARCHITECTURE.md §3`. Adding/renaming columns requires updating the consumer (step3 reads `comments_raw.csv`; `visualize.py` reads `sentiment_report.csv` + `keyword_summary.csv`).
- **Ethics**: when producing public reports, hide `authorDisplayName` (see `docs/ETHICS.md`).
